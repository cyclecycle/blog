# What to build in a world of agents

Generation will be increasingly solved. It's maybe not solved yet, but soon enough, if you can objectively verify something, if you have a way to test and measure the output, then you will be able to automate everything in between.

This means that existing apps/products that help you generate something will largely go away. It will be faster, more convenient, less friction, more accessible, more connected, and I think mostly more effective to just ask your AI to do/collaborate with you on X, be it code, images, video, audio, research, etc.

Therefore, if you're working in software/product building, and you're not building frontier AI models (as the vast majority of us are not) and you want to stick around, I think there are a few interesting areas to focus on:

- Verification
- Tools/connectivity
- Data

Verification - this is the area i'm most fascinated by. Automated verification unlocks so much for automated generation. It unlocks new training/feedback loops. It is philosophically and technically intriguing - what can you actually verify, with what confidence? 

Tools - Can you wrap some kind of 'real work' in an API that an agent can use? I.e. not something that it could easily otherwise just do itself through search and code. Good indicators are whether the tool provides unique data access OR how much time/context does it save the agent to use the tool rather than let the agent do it 'from scratch'?

Data - If you can position yourself to build/gather data with some unique value/coverage, that can be used to train/test/feed agents, that seems worth leaning into.

Of course all of these these can be seen as part of the same thing. You could have a uniquely data-enabled verification tool to expose to agents/training loops. If it works and is cost effective, you might have a business or route to acquisition in a world where everything will increasingly be agents.
